# 产品需求文档（PRD）：软件开发需求收集系统

## 1. 项目概述
**软件开发需求收集系统** 是一个交互式前端应用，旨在优化软件项目探索阶段的流程。系统通过动态分支问卷引导相关方完成技术和业务需求的梳理，在确保捕获所有关键数据的同时，为未知细节保留灵活空间。

## 2. 目标用户

- 项目经理
- 业务分析师
- 客户/项目相关方
- 开发负责人

## 3. 用户故事
- **作为用户**，我希望按照逻辑流程回答问题，避免被技术术语困扰。
- **作为用户**，我希望能够跳过不确定的问题，以便快速完成调研，同时系统会标记这些问题供后续处理。
- **作为项目相关方**，我希望看到需求的可视化摘要报告，以理解项目范围。
- **作为管理员**，我希望某些问题必须强制回答，以确保核心项目要素得到明确定义。

---

## 4. 功能需求

### 4.1 动态问卷引擎
- **分节管理：** 问题必须至少分为3个逻辑模块。
- **分支逻辑：** 特定问题的回答（例如"是否需要移动应用？"）决定后续显示的问题。
- **必填验证：** 标记为`必填`的问题必须在进入下一步前完成作答。
- **跳过逻辑：** 可选问题可以跳过；系统将此类问题标记为"待定"或"待定义"。

### 4.2 进度追踪
- 进度条显示当前模块和整体问卷的完成百分比。

### 4.3 报告生成与可视化
- **摘要视图：** 最终页面汇总所有答案。
- **可视化分析：** 使用 **Apache ECharts** 展示数据（例如项目复杂度的雷达图或功能分布的饼图）。
- **标记功能：** 未回答/跳过的问题在报告中必须高亮显示，并标注"需澄清"状态。

### 4.4 技术栈
- **框架：** SvelteKit
- **语言：** TypeScript
- **样式：** Tailwind CSS
- **测试：** Vitest
- **图表：** Apache ECharts

## 5. 问卷结构（示例数据）

系统将在3个模块中模拟以下10+个问题：

| 模块            | ID   | 问题                         | 类型 | 必填 | 逻辑/分支             |
| :-------------- | :--- | :--------------------------- | :--- | :--- | :-------------------- |
| **1. 项目核心** | Q1   | 主要平台是什么？             | 单选 | 是   | Web → Q2，移动端 → Q3 |
|                 | Q2   | 需要支持哪些浏览器？         | 多选 | 否   | -                     |
|                 | Q3   | iOS、Android还是两者都支持？ | 单选 | 是   | -                     |
| **2. 技术层面** | Q4   | 是否有偏好的技术栈？         | 布尔 | 是   | 是 → Q5，否 → Q6      |
|                 | Q5   | 列出偏好的技术               | 文本 | 是   | -                     |
|                 | Q6   | 是否需要内容管理系统？       | 布尔 | 否   | -                     |
|                 | Q7   | 预计并发用户数？             | 范围 | 否   | -                     |
| **3. 后勤规划** | Q8   | 目标上线日期是？             | 日期 | 是   | -                     |
|                 | Q9   | 是否有现成的UI设计稿？       | 单选 | 是   | 是 → Q10，否 → Q11    |
|                 | Q10  | 上传或链接设计文件           | 文本 | 否   | -                     |
|                 | Q11  | 是否需要用户体验研究服务？   | 布尔 | 否   | -                     |

## 6. UI/UX 设计规范

### 6.1 布局
- **侧边栏：** 进度导航，显示模块标题及完成状态（对勾/空心圆）。
- **主要内容区：** 居中卡片，包含当前问题、输入字段和导航按钮（返回/跳过/下一步）。
- **报告仪表板：** 清晰的网格布局，展示摘要卡片和 ECharts 图表组件。

### 6.2 待开发组件
1. `QuestionCard.svelte`：根据问题类型渲染相应的输入组件。
2. `ProgressBar.svelte`：进度的可视化指示器。
3. `ReportGenerator.svelte`：汇总答案并渲染 ECharts 的逻辑。
4. `ValidationMessage.svelte`：用于必填字段的提示消息或行内提醒。

---

## 7. 测试与质量保证

### 7.1 单元测试（Vitest）
- **目标组件：** `QuestionnaireLogic.ts`（根据分支逻辑计算下一个问题的引擎）
- **测试场景：**
    - 验证在 Q1 中选择"Web"后会跳转到 Q2。
    - 验证必填问题为空时会阻止"下一步"操作。
    - 验证被"跳过"的问题在状态中被正确记录为 `null` 或 `undefined`。

### 7.2 覆盖率目标
- 工具函数和逻辑密集型 Svelte 组件的测试覆盖率达到 80% 以上。

---

## 8. 实施计划（代码挑战策略）

1. **项目初始化：** 使用 TypeScript 和 Tailwind CSS 初始化 SvelteKit 项目。
2. **状态管理：** 创建 Svelte Store 管理问卷状态、答案和当前导航路径。
3. **问卷引擎：** 实现基于 `JSON` 结构筛选/查找下一个问题的逻辑。
4. **UI 实现：** 构建包含3个模块的表单向导。
5. **报告实现：** 集成 Apache ECharts 可视化"项目画像"（例如技术密度 vs 设计需求）。
6. **测试：** 为状态转换逻辑编写 Vitest 测试套件。
